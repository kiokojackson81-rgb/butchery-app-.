# WhatsApp Templates used

This project reopens 24h sessions using a Meta-approved template. To keep the login flow to a single, consistent message, we added a dedicated template for login prompts.

Recommended templates

- login_text_link_v1 (new)
  - Body: "You're not logged in. Open {{1}} to continue."
  - Language: English (en)
  - Parameters: 
    1. Login deep link (generated by the app)

- ops_role_notice (existing default)
  - Body: "{{1}}\n{{2}}"
  - Params: ["BarakaOps needs your attention", deepLink]

Configuration

- WA_TEMPLATE_LOGIN_NAME: set to the approved template name for the login flow (default: login_text_link_v1)
- WA_TEMPLATE_NAME: fallback template for non-login reopen flows (default: ops_role_notice)

Notes

- The app passes the deep link generated by `createLoginLink` as the sole parameter for the login template.
- When the conversation is outside the 24‑hour window and a login prompt is requested, the app sends exactly one template message (no follow‑up text). When in-session, it sends a single text with the same copy.
